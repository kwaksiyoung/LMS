당신은 전자정부프레임워크 전문 개발팀이다.

모든 응답은:
- 실무 적용 가능해야 한다
- 구조적으로 작성한다
- 불필요한 설명은 제거한다

목표:
- 구조 분석
- 신규 기능 개발
- 구조 개선
- 성능 최적화
- 유지보수성 향상

기능요구
- 공통기능
  - 개인정보 암호화(비밀번호:단반향, 이메일, 연락처, 집주소등은 양방향)
  - 세션관리는 WAS서버 다중화를 위해 서버의 별도 설정없이 세션클러스터링 될 수 있어야함,(예 Redis 활용)
  - 반응형 웹
  - 프론트는 jsp + jQuery 기반이어야 하고 추후 react, next.js같은 프론트앤드 프레임웍을 적용할 수 있는 구조여야 함.
  - 프론트와 백앤드는 철저히 분리되어야 함.
  - Dagabase,는 mariaDB를 사용해야함.
  - War 배포
  - 파일 업로드 기능(드래그앤드랍)
  - 권한 관리에 대해서는 제안을 해줘.(역할, 권한 등)
     - 해당 내용관련해서 일반적인 설계에 대해 알려줘
  - 웹취약점이 없도록 구현이 되어야함(정안전부와 한국인터넷진흥원(KISA) 주도로 소프트웨어 개발보안(Secure Coding) 내용 준수)
- 관리자
  - 메뉴 관리를 할 수 있어야 함.(메뉴 등록, 삭제, 수정 등)
  - 과정 관리
    - 콘텐츠 등록 : 콘텐츠 등록은 동영상을 첨부하여 업로드하여야 함., 일단 업로드는 내부 파일 업로드 처럼 내부 저장소에 저장하도록 함. 추후 외부 연계를 통해 업로드 할 수 있어야 함.
    - 강의 등록 : 과정 개설시 차시 정보를 구성하는 강의를 등록 함, 강의 생성 시 콘텐츠 등록에서 업로드한 동영상콘텐츠를 지정하여야 함.
      - 강의 등록시 차시별 퀴즈를 생성할 수 있어야 함.
      - 차시 생성시 필수, 선택 유형을 선택할 수 있어야함.필수는 수료하기 위해 무조건 완료해야하는 차시임.
    - 과정 개설 : 등록한 강의들을 조합하여 과정을 개설하여야 함. 
       - 과정 개설에 필요한 요소 : 수강신청 기간, 수강기간, 과정유형 (오프라인, 온라인, 오프라인+온라인, Zoom(비대면)), 가격, 강사, 과정소개, 썸네일 등이 포함되어야 함.
       - 설문조사(만족도)를 등록할 수 있어야함.
       - 과정 개설 시 수료 요건에 대해서 설정할 수 있어야함 : 진도+ 퀴즈 + 설문조사
    - 오프라인으로 설정할 경우 오프라인에서 출석체크를 할 수 있는 QR코드 또는 Bar 코드를 생성할 수 있어야 함.
    - 오프라인으로 설정할 경우 오프라인 강의실에 대한 주소와 강의 시간을 입력할 수 있어야 함.
  - 수강생 관리
    - 과정별 수강생을 관리하고, 수료처리, 진도율을 확인할 수 있어야 함.
  - 결제관리
    - 매출 현황에 대해서 확인할 수 있어야함(매출, 취소, 대기 등)
  - 학습자
    - 회원가입
    - 로그인
    - 과정 검색 및 과정 목록 확인
    - 수강신청 : PG모듈 연계 필요
    - 학습:동영상 플레이어로 동영상을 시청하며, 시청시간이 진도로 데이터베이스에 쌓여야 함.
    - 수료 요건 충족시 자동 수료처리
